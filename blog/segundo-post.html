Configuração Avançada do SiteGen

Neste post, vamos explorar as configurações avançadas do SiteGen e como personalizar seu site.

Múltiplos Templates

O SiteGen suporta múltiplos templates baseados em categorias:

[build]
templates = { 
    default = "./templates/layout.html", 
    blog = "./templates/blog.html", 
    page = "./templates/page.html" 
}

Metadados Personalizados

Você pode adicionar metadados customizados no frontmatter:

---
title: "Meu Post"
date: "2024-01-12"
category: "blog"
template: "blog"
tags: ["tag1", "tag2"]
author: "João Silva"
featured: true
---

Processamento Paralelo

O SiteGen utiliza processamento paralelo para melhor performance:

  • Detecta automaticamente o número de CPUs
  • Limita a 8 workers para evitar sobrecarga
  • Processa arquivos markdown em paralelo

Estrutura de Diretórios

Recomendamos a seguinte estrutura:

meu-site/
├── site.toml              # Configuração
├── content/               # Conteúdo markdown
│   ├── index.md
│   ├── sobre.md
│   └── blog/
│       ├── post1.md
│       └── post2.md
├── templates/             # Templates HTML
│   ├── layout.html
│   ├── blog.html
│   └── page.html
├── assets/                # Arquivos estáticos
│   ├── css/
│   ├── js/
│   └── images/
└── dist/                 # Saída gerada

Otimizações

Performance

  • Use processamento paralelo (já habilitado por padrão)
  • Minimize o uso de templates complexos
  • Otimize imagens antes de incluir

SEO

  • Adicione metadados descritivos
  • Use URLs amigáveis
  • Inclua sitemap.xml
